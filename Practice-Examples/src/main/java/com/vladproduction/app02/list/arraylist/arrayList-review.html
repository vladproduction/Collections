Commonly Used Data Structures in Java

1. Classification

  • Static: Array
  • Dynamic: ArrayList, HashMap

2. Array

  • Collection of elements stored in the heap memory
  • Fixed size and same data type
  • Can store both primitives and objects
  • Supports varargs (variable-length arguments)

Common Methods:

  • Arrays.toString()
  • Arrays.sort()
  • Arrays.binarySearch()

3. ArrayList

Java ArrayList is a resizable array implementation of the List interface. It provides dynamic arrays that can grow as needed.

Key Features:

  • Can contain duplicate elements
  • Maintains insertion order
  • Not synchronized (not thread-safe by default)
  • Allows random access via index
  • Slower than LinkedList when removing elements due to shifting
  • Does not support primitive types directly â€” use wrapper classes:
    ArrayList<Integer> list = new ArrayList<>();
  • Automatically resizes as elements are added or removed

Generic Collections:

Java generics enforce type safety and eliminate the need for casting.

ArrayList<String> list = new ArrayList<>();

If you try to add a different type, it will result in a compile-time error.

4. Vector

Vector is a legacy class that implements a dynamic array, just like ArrayList, but it is synchronized.

Example:

Vector<Integer> vector = new Vector<>();

You can also reference it via its interface:

List<Integer> list = new Vector<>();

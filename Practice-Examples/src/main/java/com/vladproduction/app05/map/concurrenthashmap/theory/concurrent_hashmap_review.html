ConcurrentHashMap in Java – Complete Guide

✅ What is ConcurrentHashMap?

A thread-safe version of HashMap that supports high concurrency and non-blocking reads introduced in Java 1.5.

📌 Key Features

  • Thread-safe without external synchronization
  • Segmented locking (Java 8 uses bucket-level synchronization)
  • No null key or values
  • High performance under multi-threaded environments

🔧 Basic Example

ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
map.put("Java", 10);
map.put("Python", 20);
System.out.println(map.get("Java"));

🔁 Iteration

for (String key : map.keySet()) {
    System.out.println(key + " → " + map.get(key));
}

❌ Remove Elements

map.remove("key");
map.remove("key", "expectedValue");

🧠 Classic Interview Problems

1. Word Frequency Counter

wordFreq.merge(word, 1, Integer::sum);

2. Concurrent Access Counter

ExecutorService exec = Executors.newFixedThreadPool(4);
exec.submit(() -> map.merge("key", 1, Integer::sum));

3. LRU Cache

Uses ConcurrentHashMap with ConcurrentLinkedDeque to maintain order.

map.put(key, value);
deque.remove(key);
deque.addFirst(key);

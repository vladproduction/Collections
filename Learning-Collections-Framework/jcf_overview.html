Introduction to Java Collection Framework

The Java Collection Framework is a unified architecture for representing and manipulating collections of objects. It provides a set of interfaces, implementations, and algorithms to work with groups of objects efficiently.

Key Benefits:
  • Reduces programming effort by providing data structures and algorithms
  • Increases performance through high-performance implementations
  • Provides interoperability between unrelated APIs
  • Reduces the effort required to learn APIs
  • Fosters software reuse

List Interface

An ordered collection that allows duplicate elements. Elements can be accessed by index.

Common List Implementations

ArrayList

Resizable array implementation. Best for frequent access operations.

List<String> list = new ArrayList<>(); list.add("Java"); list.add("Python"); list.get(0); // "Java"
Pros:
  • Fast random access O(1)
  • Memory efficient
  • Good cache locality
Cons:
  • Slow insertion/deletion O(n)
  • Fixed initial capacity

LinkedList

Doubly-linked list implementation. Best for frequent insertions/deletions.

List<String> list = new LinkedList<>(); list.add("Java"); list.addFirst("Python"); // Deque method list.addLast("C++"); // Deque method
Pros:
  • Fast insertion/deletion O(1)
  • Implements Deque interface
  • No capacity limitations
Cons:
  • Slow random access O(n)
  • Higher memory overhead

Vector

Synchronized resizable array. Thread-safe but slower performance.

List<String> list = new Vector<>(); list.add("Java"); // All methods are synchronized
Pros:
  • Thread-safe
  • Similar to ArrayList
Cons:
  • Performance overhead
  • Legacy class

Set Interface

A collection that contains no duplicate elements. Models mathematical set abstraction.

HashSet

Hash table implementation. No ordering guarantees.

Set<String> set = new HashSet<>(); set.add("Java"); set.add("Python"); set.add("Java"); // Duplicate - won't be added // Size is 2
Pros:
  • Fast O(1) operations
  • No duplicates
  • Memory efficient
Cons:
  • No ordering
  • Requires good hashCode()

LinkedHashSet

Hash table with linked list. Maintains insertion order.

Set<String> set = new LinkedHashSet<>(); set.add("Java"); set.add("Python"); set.add("C++"); // Iteration order: Java, Python, C++
Pros:
  • Maintains insertion order
  • Fast O(1) operations
  • No duplicates
Cons:
  • Higher memory overhead
  • Slightly slower than HashSet

TreeSet

Red-Black tree implementation. Sorted order.

Set<String> set = new TreeSet<>(); set.add("Python"); set.add("Java"); set.add("C++"); // Iteration order: C++, Java, Python (sorted)
Pros:
  • Sorted order
  • NavigableSet operations
  • O(log n) operations
Cons:
  • Slower than HashSet
  • Requires Comparable elements

Queue Interface

A collection designed for holding elements prior to processing. Typically FIFO order.

PriorityQueue

Heap-based priority queue. Elements ordered by priority.

Queue<Integer> pq = new PriorityQueue<>(); pq.offer(5); pq.offer(1); pq.offer(3); pq.poll(); // Returns 1 (smallest)

ArrayDeque

Resizable array implementation of Deque. Can be used as stack or queue.

Deque<String> deque = new ArrayDeque<>(); deque.addFirst("First"); deque.addLast("Last"); deque.removeFirst(); // "First"

Map Interface

An object that maps keys to values. Cannot contain duplicate keys.

HashMap

Hash table implementation. Fast O(1) operations.

Map<String, Integer> map = new HashMap<>(); map.put("Java", 1995); map.put("Python", 1991); map.get("Java"); // 1995

LinkedHashMap

Hash table with linked list. Maintains insertion or access order.

Map<String, Integer> map = new LinkedHashMap<>(); map.put("Java", 1995); map.put("Python", 1991); // Maintains insertion order

TreeMap

Red-Black tree implementation. Sorted by keys.

Map<String, Integer> map = new TreeMap<>(); map.put("Python", 1991); map.put("Java", 1995); // Keys sorted: Java, Python

Best Practices and Tips

1. Choose the Right Collection

  • ArrayList: When you need fast random access
  • LinkedList: When you do frequent insertions/deletions
  • HashSet: When you need fast lookup and no duplicates
  • TreeSet: When you need sorted unique elements
  • HashMap: When you need fast key-value lookup
  • TreeMap: When you need sorted key-value pairs

2. Programming to Interfaces

// Good List<String> list = new ArrayList<>(); Set<String> set = new HashSet<>(); Map<String, Integer> map = new HashMap<>(); // Avoid ArrayList<String> list = new ArrayList<>(); HashSet<String> set = new HashSet<>();

3. Initial Capacity

// If you know approximate size List<String> list = new ArrayList<>(1000); Set<String> set = new HashSet<>(1000); Map<String, Integer> map = new HashMap<>(1000);

4. Immutable Collections (Java 9+)

List<String> list = List.of("Java", "Python", "C++"); Set<String> set = Set.of("Java", "Python", "C++"); Map<String, Integer> map = Map.of( "Java", 1995, "Python", 1991 );

5. Thread Safety

// Synchronized wrappers List<String> syncList = Collections.synchronizedList(new ArrayList<>()); Set<String> syncSet = Collections.synchronizedSet(new HashSet<>()); // Concurrent collections Map<String, Integer> concurrentMap = new ConcurrentHashMap<>();

6. Iteration Best Practices

// Enhanced for loop (preferred) for (String item : list) { System.out.println(item); } // Streams (Java 8+) list.stream() .filter(s -> s.startsWith("J")) .forEach(System.out::println); // Iterator for safe removal Iterator<String> it = list.iterator(); while (it.hasNext()) { if (condition) { it.remove(); // Safe removal } }

Key Takeaways:

  • Always program to interfaces, not implementations
  • Choose collections based on your use case and performance requirements
  • Consider thread safety requirements early in design
  • Use generics for type safety
  • Understand the performance characteristics of each collection
  • Use factory methods for creating immutable collections when possible
  • Always override equals() and hashCode() for custom objects used as keys
